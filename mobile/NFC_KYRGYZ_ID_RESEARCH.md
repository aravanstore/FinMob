# Отчет об интеграции NFC для Кыргызских ID-карт (2024)

## 🎯 Цель
Автоматизировать регистрацию клиентов путем чтения данных (ФИО, Адрес) напрямую с чипа ID-карты через NFC.

## ✅ Что реализовано и работает (Verified)
1.  **Обнаружение чипа (AID Discovery):**
    *   Подтверждено приложение: `ePassport (Std)` с AID `A0 00 00 02 47 10 01`.
    *   Чип отвечает по протоколу **ISO 14443-4** (Java Card).
2.  **Алгоритм формирования ключей (ICAO 9303):**
    *   Реализован расчет контрольных цифр (Check Digits).
    *   **Пример рабочего ключа (BAC):** `ID2699802376083023203259`. Математика совпадает с профессиональными калькуляторами MRZ.
3.  **OCR (Распознавание текста):**
    *   Стабильное чтение номера паспорта и дат с обеих сторон карты.
    *   Автоматическая транслитерация латиницы в кириллицу.

## ❌ Что не удалось (Blockers)
1.  **Авторизация (BAC):**
    *   Чип возвращает ошибку `69 82` (Security condition not satisfied) при попытке чтения файлов DG1 и DG13.
    *   Самодельная реализация BAC-хендшейка на Dart (3DES) не прошла проверку чипом.
2.  **Доступ к адресу (DG13):**
    *   Файл найден, но заблокирован. Прямой доступ без сессионных ключей невозможен.

## 🧬 Спецификация чипа (по данным TagInfo)
*   **Производитель:** Samsung Electronics (IC Fabricator).
*   **Платформа:** Java Card version 2.2 / Global Platform 2.1.1.
*   **Дата выпуска ОС:** 19 марта 2024 года (свежая прошивка).
*   **Доступные приложения (AIDs):**
    *   `Global Platform card manager` (управление).
    *   `Electronic identity document (MRTD)` — целевое приложение ICAO.
    *   `Visa card manager` — банковский слой.
*   **Технологии:** ISO/IEC 14443-4 (Type A), Mifare Classic (1kB emulation).

## 🔍 Технические выводы (Обновлено)
*   **Причина неудачи BAC:** Современные чипы Samsung 2024 года после Mutual Authentication требуют обязательного **Secure Messaging (SM)**. Любая попытка прочитать DG1/DG13 в открытом виде (как делали мы) приводит к ошибке `69 82` или разрыву соединения.
*   **PACE:** Наличие Global Platform 2.1.1 и свежей даты выпуска ОС делает PACE практически обязательным для доступа к данным.

## 🚀 Рекомендация для следующего этапа
Прекратить «ручное» тестирование через APDU-команды. 
**Необходимо интегрировать профессиональную библиотеку на Dart:**
*   **[vcmrtd](https://github.com/privacybydesign/vcmrtd)** — поддерживает ICAO 9303, BAC и PACE.
*   Использовать уже вычисленную нами строку MRZ в качестве `BACKey` в этой библиотеке.
