# Заметка: Бэкенд FinMob (Meridian Mobile API)

## Основная информация
*   **Локальный порт:** `3002`
*   **Команда для запуска:** `npm run dev` (в папке `backend`)
*   **Главная база данных (Control DB):** `fincore_control` (прописана в `.env` как `PG_CONTROL_DB`). Именно здесь таблица `tenants` хранит доступы.
*   **Базы организаций (Tenants):** Названия баз для конкретных организаций (например, `boy`, `meridian`). Приложение запрашивает доступ по коду организации.

## Подключение из приложения
*   Для **Android-эмулятора**: `http://10.0.2.2:3002` (плюс команда `adb reverse tcp:3002 tcp:3002` для надежности).
*   Для **VS Code / Android Studio**: `devUrl` сейчас установлен на `http://10.0.2.2:3002`.

## Как работает маршрутизация к БД
1. Пользователь вводит код организации (например, `boy`), телефон и ПИН.
2. Бэкенд ищет `boy` в таблице `tenants` внутри базы `fincore_control`.
3. Если находит, бэкенд подключается к отдельной базе данных с именем `boy` и уже там сверяет ПИН пользователя в таблице `mobile_client_access`.
